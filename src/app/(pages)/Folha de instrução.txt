​Guia de Desenvolvimento - Projeto de Estudo

​Bem-vindos ao nosso projeto! Para mantermos o código organizado e evitar conflitos, sigam estas diretrizes básicas:

​1. Onde você PODE mexer

​Pasta src/app/(pages)/: Este é o nosso playground. Podem criar novas pastas e arquivos aqui para desenvolver as páginas do projeto.
​Pasta public/: Para adicionar imagens, fontes ou arquivos estáticos.

​2. Onde você NÃO DEVE mexer
​src/app/layout.js: Este arquivo controla a estrutura global do site.
​src/app/page.js (Raiz): Este arquivo serve apenas para redirecionar o usuário para a página inicial. A única alteração permitida aqui é o nome da rota no comando redirect.

​Configurações: Arquivos como next.config.mjs, jsconfig.json e package.json devem ser alterados apenas com supervisão.

​3. Como criar uma nova página

​Para criar uma nova tela (exemplo: "Dashboard"):
​Crie uma pasta dentro de src/app/(pages)/ com o nome da rota (ex: dashboard).
​Dentro dessa pasta, crie dois arquivos:
​page.js (O arquivo principal da página).
​page.module.css (Para os estilos exclusivos desta página).
​
4. Estrutura Padrão do page.js

​Utilizem sempre este modelo para garantir a consistência:

-------
import styles from './page.module.css';

export default function NomeDaPagina() {
  return (
    <main>
      {/* Seu código HTML aqui */}
    </main>
  );
}
-------

5. Como alterar o redirecionamento inicial
​Se você estiver desenvolvendo uma nova funcionalidade e quiser que o site abra nela primeiro, vá ao arquivo src/app/page.js (na raiz da pasta app) e altere o caminho dentro do redirect:

// Altere apenas o texto entre aspas
redirect('/nome-da-sua-nova-pagina');
